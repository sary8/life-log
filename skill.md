# Life Log

話し言葉の入力をカテゴリ分けされたマークダウン日記にまとめ、`daily/YYYY/MM/DD.md` に保存する。ディレクトリがなければ作る。保存後はコミット・プッシュまで行う。

## フォーマット

markdownlint準拠。見出し・リスト・水平線の前後に空行。ファイル末尾は改行。

```markdown
# 2026-03-12（木）

## ⚖️ 体重

- 75.8kg / 体脂肪率 20.9%

## 🍽️ 食事

- 朝：鶏むね肉そぼろ、キムチ、ポーチドエッグ
- 昼：弁当（鶏そぼろ）
- 夜：友達と渋谷で焼肉

## 🏋️ 健康

- ジムで筋トレした

## 💼 仕事

- 会議を3件こなした

## 👥 プライベート

- 大学の友達と渋谷でディナー

## 💭 振り返り

会議が多くて疲れたけど、夜の食事でリフレッシュできた。

---

tags: #仕事 #ジム #友人 #渋谷
```

## カテゴリ

この順で並べる。該当なしは省略。振り返りは常に最後。

⚖️ 体重 → 🍽️ 食事 → 🏋️ 健康 → 🍳 料理 → 💼 仕事 → 📚 学習 → 👥 プライベート → 🎮 趣味 → 🏠 生活 → 💰 お金 → ✈️ 旅行 → 💭 振り返り

一覧にないカテゴリは適当な位置に挿入。1カテゴリでも問題ない。

### 体重・食事・料理の区別

- **体重** — `XX.Xkg / 体脂肪率 XX.X%` の固定フォーマット。言及がない日は `未計測`
- **食事** — 食べたものの記録。「朝：」「昼：」「夜：」で分ける。未言及の時間帯は書かない
- **料理** — 調理行為の記録。仕込み・作り置き等。食べた記録とは別

## 日付

`# YYYY-MM-DD（曜）` 形式。曜日は正しく計算する。指定がなければ当日。「昨日」等の相対指定は解釈する。

## 追記

同日に複数回呼ばれたら既存ファイルに追記。

- 既存カテゴリ → 箇条書き追加
- 新カテゴリ → 順番に従って挿入
- 振り返り → 一日全体を踏まえて書き直す
- タグも更新。重複は書かない

## タグ

ファイル末尾 `---` の後に `tags: #タグ1 #タグ2`。内容から3〜6個。

## 入力の解釈

話し言葉から適切なカテゴリに振り分ける。ユーザーの言葉を過度にフォーマルにしない。言っていないことは補完しない。振り返りはユーザーの感情を汲み取って自然に書く。

> 「朝ジム行ってから仕事行って、会議3つもあってさ、疲れたわ。でも夜は友達と飯食って楽しかった」

→ ジム=健康、会議=仕事、友達と飯=プライベート、疲れた/楽しかった=振り返り
